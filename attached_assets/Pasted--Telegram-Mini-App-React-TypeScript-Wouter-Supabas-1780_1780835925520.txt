Задача: доработать Telegram Mini App для оценки сотрудников на React + TypeScript + Wouter + Supabase.

Текущая ситуация:

* Оценки сохраняются в таблицу evaluation_results.
* Есть нижняя навигация:

  * Оценка
  * История
  * Аналитика
  * Настройки
* В Dashboard уже есть аналитика по сотрудникам.
* В поле answers теперь хранится массив объектов:

[
{
"question": "Текст вопроса",
"score": 2,
"comment": ""
}
]

Не ломать существующую логику оценки и сохранения.

Не менять структуру базы данных без необходимости.

Что нужно реализовать:

1. Раздел "Аналитика"

Сделать внутри вкладки:

* Сотрудники
* Вопросы
* Чек-листы
* История

По умолчанию открывать "Сотрудники".

---

2. Аналитика сотрудников

Оставить текущую реализацию.

Показывать:

* имя сотрудника
* средний балл
* количество оценок

По нажатию открывать детализацию сотрудника.

---

3. Аналитика вопросов

Использовать данные из evaluation_results.answers.

Для каждого вопроса отображать:

* текст вопроса
* средний балл
* количество ответов

Отсортировать по среднему баллу от меньшего к большему.

Пример:

Обратная связь
Средний балл: 1.7
Ответов: 12

---

4. Аналитика чек-листов

Группировать данные по checklist_name.

Для каждого чек-листа показывать:

* название чек-листа
* средний балл
* количество оценок

Пример:

Оценка Team Leader

Средний балл: 2.4
Оценок: 15

---

5. История

Перенести существующий экран истории во вкладку История внутри раздела Аналитика.

Сохранить переход в детали оценки.

---

6. Главный Dashboard

Добавить сверху KPI карточки:

* Всего оценок
* Средний балл по системе
* Количество сотрудников с оценками

---

7. Фильтры

Для аналитики добавить фильтрацию:

* по сотруднику
* по чек-листу
* по оценщику

Фильтры должны работать для всех вкладок аналитики.

---

8. Подготовка к требованиям брифа

Структура аналитики должна поддерживать:

* динамику оценок сотрудника
* графики по месяцам
* экспорт данных

Но сами графики пока не реализовывать.

---

9. UI

Использовать существующий стиль приложения:

* Card
* CardContent
* Tailwind
* мобильный интерфейс
* Telegram Mini App стиль

Не использовать сторонние библиотеки для графиков на этом этапе.

---

Результат:

После выполнения должен существовать единый раздел "Аналитика", содержащий:

* Сотрудники
* Вопросы
* Чек-листы
* История

и полностью закрывающий аналитические требования брифа.
